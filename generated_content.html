Software Engineering Q&A

1. What are the three characteristics of Software?

Software is engineered not manufactured, software doesn't wear out and software is custom built

Simplified Explanation:

Software encompasses:

  1. Instructions (computer programs) that when executed provide desired features, function, and performance;
  2. Data structures that enable the programs to adequately store and manipulate information.
  3. Documentation that describes the operation and use of the programs.

2. What is the definition of a Process in software engineering?

A process is a collection of activities, actions and tasks that are performed when some work product is to be created

Simplified Explanation:

In software engineering, a 'process' is a structured set of activities, actions, and tasks designed to create a software product. This ensures software development is controlled and efficient. A well-defined process provides a roadmap from idea to finished software.

3. What are the primary goals of Software Engineering?

  • To improve the quality of software products
  • To increase the productivity
  • To give job satisfaction to the software engineers

Simplified Explanation:

Software Engineering focuses on creating reliable and efficient software. It involves understanding user needs, planning, designing, building, and deploying software. Key goals are high-quality software, increased productivity for developers, and job satisfaction for engineers.

4. What are the concerns about applying the Formal Methods Model in a business environment?

  • The development of formal models is currently time consuming and expensive
  • Few developers have the required formal methods background
  • The models are difficult to use as a communication mechanism for non-technical customers

Simplified Explanation:

Concerns about using Formal Methods in business:

  • **Expensive:** Requires specialized skills and takes longer.
  • **Steep Learning Curve:** Most developers need training.
  • **Communication Gap:** Hard to explain to non-technical people.
